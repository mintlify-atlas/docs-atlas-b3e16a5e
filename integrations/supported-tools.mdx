OpenSpec integrates with 24 AI coding assistants. When you run `openspec init`, OpenSpec configures your selected tools by installing skills and commands based on your active profile.

## How it works

For each selected tool, OpenSpec installs two types of integration files:

1. **Skills** — Markdown files placed in the tool's skills directory (e.g., `.claude/skills/openspec-*/SKILL.md`). AI coding assistants auto-detect these files and load them as context.
2. **Commands** — Tool-specific slash command files (e.g., `.cursor/commands/opsx-propose.md`) that wire up `/opsx-*` commands in the tool's interface.

Both are generated based on your selected profile and kept in sync when you run `openspec update`.

## Default profile

By default, OpenSpec uses the `core` profile, which installs four workflows:

| Workflow | Command |
|---|---|
| `propose` | `/opsx:propose` |
| `explore` | `/opsx:explore` |
| `apply` | `/opsx:apply` |
| `archive` | `/opsx:archive` |

## Enabling expanded workflows

To install additional workflow commands (`new`, `continue`, `ff`, `verify`, `sync`, `bulk-archive`, `onboard`), configure your profile and refresh:

```bash
openspec config profile
openspec update
```

This updates all generated skills and command files to match your new selection.

## Delivery modes

OpenSpec supports three delivery modes per tool:

| Mode | What gets installed |
|---|---|
| **Skills only** | Only `SKILL.md` files in the tool's skills directory |
| **Commands only** | Only slash command files |
| **Both** (default) | Both skills and command files |

Most tools use both. The `trae` tool uses skills only (no command adapter exists).

## Non-interactive setup

For CI/CD or scripted environments, pass flags directly to `openspec init`:

```bash
# Configure specific tools
openspec init --tools claude,cursor

# Configure all supported tools
openspec init --tools all

# Skip tool configuration
openspec init --tools none

# Override profile for this init run
openspec init --profile core
```

**Available tool IDs:** `amazon-q`, `antigravity`, `auggie`, `claude`, `cline`, `codex`, `codebuddy`, `continue`, `costrict`, `crush`, `cursor`, `factory`, `gemini`, `github-copilot`, `iflow`, `kilocode`, `kiro`, `opencode`, `pi`, `qoder`, `qwen`, `roocode`, `trae`, `windsurf`

## Generated skill names

Depending on your profile selection, OpenSpec generates some or all of these skills:

- `openspec-propose`
- `openspec-explore`
- `openspec-new-change`
- `openspec-continue-change`
- `openspec-apply-change`
- `openspec-ff-change`
- `openspec-sync-specs`
- `openspec-archive-change`
- `openspec-bulk-archive-change`
- `openspec-verify-change`
- `openspec-onboard`

Skill and command counts are profile-dependent. The `core` profile installs 4 skills; enabling all workflows installs all 11.
