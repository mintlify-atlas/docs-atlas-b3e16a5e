OpenSpec adds a lightweight spec layer between your ideas and your AI coding assistant. Instead of vague prompts leading to unpredictable results, you and your AI align on requirements, design, and tasks — then implement with confidence.

```text
→ fluid not rigid
→ iterative not waterfall
→ easy not complex
→ built for brownfield not just greenfield
→ scalable from personal projects to enterprises
```

## How it works

<Steps>
  <Step title="Propose a change">
    Run `/opsx:propose add-dark-mode` in your AI assistant's chat. OpenSpec creates a change folder with all planning artifacts.
  </Step>
  <Step title="Implement tasks">
    Run `/opsx:apply` to work through the task checklist. The AI writes code guided by the specs and design you agreed on.
  </Step>
  <Step title="Archive the change">
    Run `/opsx:archive` to merge delta specs into your source of truth and preserve the full history.
  </Step>
</Steps>

```text
You: /opsx:propose add-dark-mode
AI:  Created openspec/changes/add-dark-mode/
     ✓ proposal.md — why we're doing this, what's changing
     ✓ specs/       — requirements and scenarios
     ✓ design.md    — technical approach
     ✓ tasks.md     — implementation checklist
     Ready for implementation!

You: /opsx:apply
AI:  Implementing tasks...
     ✓ 1.1 Add theme context provider
     ✓ 1.2 Create toggle component
     ✓ 2.1 Add CSS variables
     ✓ 2.2 Wire up localStorage
     All tasks complete!

You: /opsx:archive
AI:  Archived to openspec/changes/archive/2025-01-23-add-dark-mode/
     Specs updated. Ready for the next feature.
```

## Why OpenSpec?

AI coding assistants are powerful but unpredictable when requirements live only in chat history. OpenSpec solves this with a structured change management layer that keeps humans and AI aligned.

  <Card title="Agree before you build" icon="handshake">

  </Card>
  <Card title="Stay organized" icon="folder-open">

  </Card>
